🧩 Background

I worked with the development team of SWIL enterprise with multiple SAAS Products—like inventory systems, invoicing tools, CRM dashboards, and mobile apps. All of them were built at different times, by different teams, and with no shared design approach.

This led to:

  • •Inconsistent UI elements (e.g., 3 types of buttons across apps)
  • •Different UX patterns (e.g., filters on top in one app, on side in another)
  • •Increased dev time for every small change
  • •Confused users, especially those working across multiple products

Product Portfolio Overview

Inventory Systems Different button styles and patterns
Invoicing Tools Inconsistent form layouts
CRM Dashboards Scattered filter placements
Mobile Apps No shared design approach

Our Goal

Create a shared design system that addresses all consistency and efficiency challenges across the enterprise products.

UI/UX Consistency

Bring consistency in UI/UX across all products with unified design patterns and components

Developer Efficiency

Make it easy to implement for developers with reusable components and clear documentation

Fast Implementation

Work within tight deadlines while maintaining quality and ensuring smooth adoption

Enhanced User Experience

Improve user experience without breaking existing flows and familiar patterns

Design System Creation Process

1

Understanding What Exists

1. Audit of All Products

I started by reviewing every live product and listing out all design patterns—buttons, dropdowns, modals, tables, etc.

🔧 Tools: Figma (screenshots), FigJam (mapping inconsistencies)

📌 What I found:
  • • 5+ styles of primary buttons
  • • Different font sizes used randomly
  • • No predictable behavior across apps

2. Talking to Users & Teams

I interviewed internal users (sales, ops) and talked to product managers, devs, and support teams.

Key insights:
  • • Users had to "relearn" each product
  • • Devs wasted time recreating components
  • • PMs wanted more speed but couldn't risk usability
2

Planning the Design System

To avoid overcomplicating things, I focused on creating a design system, not just a component library.

✅ What we defined:

  • Design Tokens: Colors, spacing, typography
  • Components: Buttons, inputs, tables, modals, tabs, etc.
  • UX Patterns: Form handling, loading states, empty states
  • Naming & Folder Rules: For easy handoff

UX Principles Applied:

  • Jakob's Law: Keep patterns familiar for users switching between apps
  • Fitts' Law: Improved size/spacing for mobile and desktop users
  • Law of Proximity: Grouped related actions together
  • Hick's Law: Reduced too many choices in dropdowns and filters
3

Collaboration & Building It

1. Design + Dev Sync

  • • Weekly meetings with devs to align naming, behavior, and responsiveness
  • • Created Before vs After boards to explain reasoning
  • • Used Figma Variants + Auto Layout to build scalable components

2. Quick Feedback Loop

  • • Shared clickable prototypes with internal users
  • • Took feedback early to avoid big rewrites
  • • PMs loved seeing "Why we made this change" notes with each component
4

Fast Delivery with AI Prototypes

We didn't have time to wait for a coded design library, so I proposed using AI-based tools for faster developer handoff.

What I did:

  • • Built shareable prototype file with documentation using Figma Make AI
  • • Auto-generated basic code previews (React)(Bootstrap)
  • • Developers could inspect interactions, variants, and spacing Guide

This helped us skip the extra dev-design meetings and allowed faster implementation across products.

🧭 Key Learnings

🎯
Simplicity Over Complexity: Simplicity wins over complexity in fast-paced teams. Focused approach on essential components and patterns yielded better adoption.
🤝
Early Stakeholder Involvement: Involving developers and users early in the design system creation process makes adoption easier and reduces resistance to change.
📈
Design System Beyond Design: A design system is never just about design—it's about scaling good experience and creating shared understanding across teams.
⚡
AI-Powered Efficiency: Using AI tools for rapid prototyping and code generation significantly reduced handoff time and improved developer experience.

🙌 Final Thoughts

This project taught me that creating a successful design system requires more than just designing components—it's about building bridges between teams, establishing shared understanding, and creating sustainable processes. The combination of systematic thinking, collaborative approach, and modern AI tools enabled us to deliver a scalable solution that truly transformed how our enterprise approaches product design. Most importantly, seeing teams adopt and extend the system organically proved that when you solve real problems with simple, well-documented solutions, adoption happens naturally.
